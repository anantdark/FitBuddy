Usage guide

After onboarding, FitBuddy has three tabs — Dashboard, Progress, and Body — plus a Log button and Settings.

Main tabs

  • Dashboard — calorie ring, macros remaining, and the food / exercise log for the selected day. Empty state: Tap Log to add food or a workout.
  • Progress — Weekly or Monthly charts (net calories, macros, body composition, burn) and the AI Progress Coach.
  • Body — profile basics, body readings, daily targets, food library.

On Dashboard, the floating Log button opens the log hub. The gear opens Settings.

Browse & log past days

The Dashboard week strip shows the last seven days. Tap a day pill to jump there — the ring, macros, and log update for that date. Labels like Today / Yesterday make it easy to spot where you are.

  • Week history — open the week view for stacked calorie bars and that day’s entries; swipe weeks to look further back.
  • Log onto a past day — select the day first, then use Log. New meals and workouts are stamped on that calendar day (not always “now”).
  • Clone to today — from an older entry you can copy a meal or workout onto real today when you want to repeat it.

Log food

From Log, under Food:

  • Photo — take a picture of the plate
  • Gallery — pick an existing image
  • Describe — freeform text (Food or activity → Analyze)
  • Build meal — assemble multiple foods
  • Saved meal — log a meal preset in one tap

Build a meal

Opens Build a meal. Add items via Scan (barcode), Saved food, Create new food, or Add food manually, then Review & save meal.

Barcode

Point the camera at a barcode, or type an EAN/UPC. FitBuddy looks up the product on Open Food Facts (needs network). From Body you can also Scan barcode to save into your food library.

Review & edit

After AI analysis you get Review food (or a meal review). Typical actions:

  • Edit dish name and ingredient weights — macros rescale live
  • Recalculate with AI if the estimate looks off
  • Bookmark: Save to food library (single food) or Save as meal preset (meal)
  • Confirm: Log food / Save meal / Update meal

From the selected day’s log you can open an entry again to edit or delete it.

Log a workout

Log → Workout opens Log Workout. Add exercises (sets, reps, weight, or duration/distance as needed), then Save & Estimate Calories. FitBuddy uses AI (when configured) plus your body factors for the burn estimate. Existing sessions can be opened as Edit Workout and saved with Save Changes.

Body tab

Under Body & goals:

  • Body basics — age, height, weight, sex, activity, goal
  • Body composition — Add a reading (weight and optional metrics)
  • Daily targets — calories and macros; optional Recommend with AI → review the plan → Apply or Discard
  • Food library — saved foods; manage or scan new ones
  • Save body data when you edit fields manually
  • Reading history — past measurements

Progress & coach

Toggle Weekly / Monthly. In Monthly mode, use the month arrows to step through history (e.g. This month · Jul 2026). Charts include net calories vs target, macronutrient trend, body composition, and calories burned.

AI Progress Coach: Generate insight / Refresh insight, then Ask follow-up questions to open Progress Coach chat. The coach now gets a compact multi-month snapshot (recent days plus older monthly averages) so insights stay useful as your history grows. Leaving may ask Leave conversation? with Keep chat or Clear & go back.

Coach features need a configured AI provider (same as logging).

Settings

AI Provider

  • Auto failover — on failure, try other API keys, then other models on the same platform. Never switches platforms automatically. When off, only the selected model is used; keys still rotate on failure.
  • Provider segments: OpenRouter / Gemini / Ollama (Local or Cloud)
  • Multiple API keys as chips
  • Photo model and Text model dropdowns
  • Save AI Settings

Other

  • Material You — dynamic color on Android 12+
  • Backup & Data — Export / Import JSON
  • Send crash reports and a copyable Support ID (see Privacy)
  • Check automatically for updates, or Check for Updates manually

Install & updates

  1. Download the APK from GitHub Releases.
  2. Allow install from your browser/file manager if Android asks.
  3. Later, Settings (or automatic check) can show Update available → Download & install. Keep the app open until the system installer opens.

Same signing key required for in-app updates.

Official GitHub release APKs share one release keystore. Mixing a locally debug-signed build with a release update can fail to install over the existing app.
