Onboarding

First you choose how to start — fresh install or restore a backup — then FitBuddy walks you through AI setup and your profile. An AI connection is required; there is no skip.

How do you want to start?

On a new install you’ll see a path picker before the three setup steps:

  • Start Fresh — mint a new Support ID and go through AI + profile setup.
  • Restore from cloud backup — enter the Support ID from the device that uploaded the backup (same ID shown under Settings → Backup). Available when this build includes cloud backup.
  • Restore from local backup — pick a JSON file you previously exported.

After a successful restore, if AI credentials are missing or no longer work, FitBuddy opens Reconnect AI so you can connect a provider again without re-entering your whole profile.

Keep your Support ID safe.

Cloud backups are keyed by that ID. Copy it from Settings → Backup on the old device before you wipe or reinstall.

Step 1 — Connect an AI

FitBuddy uses a large language model to read food photos, freeform text, and to design your initial calorie targets. Choose a provider and add credentials before you can continue.

  • OpenRouter — tap Continue with OpenRouter to sign in (no API key paste required), or expand Use API keys instead to add one or more keys manually. If both are set, manual keys are preferred; the OpenRouter sign-in is used only if they fail.
  • Gemini — Google AI Studio API key(s)
  • Ollama — Local (server URL, e.g. a LAN host) or Cloud (Ollama Cloud API key)

Tap Continue. The app validates by listing models from that provider (Validating… / Connecting… during OpenRouter sign-in). If the key, URL, or sign-in fails, you’ll see an error such as Couldn’t validate that API key and stay on this step.

Your credentials stay on-device.

Keys and the OpenRouter OAuth token only leave the phone when FitBuddy calls the provider you selected. Docs links on this screen open the provider’s key-creation guides in the browser.

Step 2 — About you

Enter the basics used for calorie and workout estimates:

  • Age
  • Height (cm)
  • Current weight (kg)
  • Sex — Prefer not to say / Male / Female

Continue unlocks when age, height, and weight are in a sensible range.

Step 3 — Your lifestyle

  • Activity level — Sedentary → Very active
  • Goal — Let AI decide / Lose weight / Gain muscle / Body recomposition

Tap Finish setup (Setting up… while saving).

After Finish setup

  1. AI settings and your profile are saved locally.
  2. An initial body weight reading is recorded.
  3. The main dashboard opens.
  4. FitBuddy requests an AI calorie/macro plan. You’ll see Calculating your calorie targets…, then Your personalized targets are ready (or defaults with a note to refine anytime in Body).

You can change providers, keys, models, reminders, and backup later under Settings (gear icon). Targets and body data live under the Body tab.

Getting an API key

  • OpenRouter — easiest: use Continue with OpenRouter in the app. Or create a key via Authentication docs
  • Gemini — API key guide
  • Ollama — Install & setup · Cloud keys: ollama.com/settings/keys
